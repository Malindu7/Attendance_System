Row 1 — Image Preparation (Stages 1–3)

1Raw BGR Frame

OpenCV captures frames from the webcam in BGR (Blue-Green-Red) channel order — the raw unprocessed input from the camera sensor.

cv2.VideoCapture(0, cv2.CAP_DSHOW)

2BGR → RGB Conversion

face_recognition and ResNet-34 expect RGB format. Without this conversion the model reads wrong colour channels and generates completely wrong face encodings.

cv2.cvtColor(frame, COLOR_BGR2RGB)

3CLAHE Lighting

Contrast Limited Adaptive Histogram Equalisation in LAB colour space. Only the L (luminance) channel is equalised across local 8×8 tiles. Fixes dark, backlit and uneven classroom lighting without distorting skin tones.

createCLAHE(clipLimit=2.0, tileGridSize=(8,8))

Row 2 — Noise Removal & Face Detection (Stages 4–6)

4Gaussian Blur

A 3×3 Gaussian kernel with σ=0.5 removes random pixel-level sensor noise. Prevents false gradient edges from disrupting HOG detection without blurring facial features.

GaussianBlur(frame, (3,3), sigmaX=0.5)

5Grayscale Conversion

HOG detection works on intensity gradients, not colour. Converting to a single grayscale channel speeds up gradient computation by 3× and reduces memory usage.

Luminance = 0.299R + 0.587G + 0.114B

6HOG Face Detection

Histogram of Oriented Gradients — 5 stages: compute gradients → build 9-bin cell histograms → L2 block normalisation → SVM classification → image pyramid. Frame resized to 25% first for real-time speed.

face_locations(small, model='hog')

Row 3 — Face Extraction & Preparation (Stages 7–9)

7ROI Crop

The detected bounding box is cropped from the frame with 20px padding on all sides. This isolates only the face region for encoding — removing background clutter that would confuse the CNN.

face_region = frame[top-20:bottom+20, left-20:right+20]

8Face Alignment

68 facial landmarks detected. Left and right eye centres computed. Rotation angle calculated using arctan2. A rotation matrix is applied so the face is horizontally level. The cyan line on the panel shows the eye axis and alignment angle.

angle = arctan2(dy, dx) → getRotationMatrix2D → warpAffine

9Pixel Normalisation

Each pixel divided by 255 to bring values to 0–1 range. Then ImageNet mean (0.485, 0.456, 0.406) is subtracted and divided by std (0.229, 0.224, 0.225). Displayed as a plasma heatmap showing intensity distribution.

norm = (pixel/255 − mean) / std

Row 4 — Encoding & Matching (Stages 10–11)

10ResNet-34 Encoding

The normalised face passes through 34 convolutional layers of a ResNet pretrained on 3 million faces using triplet loss. Output is a 128-dimensional float64 vector — a unique mathematical fingerprint. The bar chart shows all 128 values with colour representing magnitude.

face_encodings(frame, locations) → [128 float64 values]

11Euclidean Distance Match

The 128-D vector is compared against all registered face encodings using Euclidean distance. Distance < 0.50 = same person → VERIFIED → logged to MongoDB once per day. The cyan line on the distance bar shows the 0.50 threshold. Green = match, blue = unknown.

face_distance(known_encodings, encoding) < 0.50

💡Why 11 stages?

Each stage solves a specific real-world problem. CLAHE handles classroom lighting variation. Gaussian blur removes camera noise. HOG is fast enough for real-time use. ResNet-34 is robust to changes in appearance. Together they achieve reliable automated attendance.
